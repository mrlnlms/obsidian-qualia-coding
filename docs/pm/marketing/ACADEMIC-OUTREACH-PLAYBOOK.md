### Conference Submission Template (500-word abstract)

> **Title:** Qualia Coding: Qualitative Data Analysis as Knowledge Management Inside Obsidian
>
> **Background:** Qualitative researchers increasingly use personal knowledge management (PKM) tools alongside CAQDAS for interpretive synthesis (Silver & Lewins, 2014). However, the separation between coding environments and thinking environments creates friction: manual excerpt transfer, lost provenance, and fragmented workflows.
>
> **Innovation:** Qualia Coding is a free, open-source Obsidian plugin that unifies qualitative coding and knowledge management in a single environment. It provides seven coding engines (Markdown, PDF, CSV/Parquet, Image, Audio, Video) with a shared codebook, unified sidebar, and 17 analytics views — including MCA Biplot, MDS Map, Lag Sequential Analysis, Polar Coordinates, and CHAID Decision Trees. These analytical capabilities, typically requiring R or SPSS, are built into the coding tool itself, enabling crossover mixed analysis (Onwuegbuzie & Johnson, 2021) without tool-switching.
>
> **Design Principles:** Grounded in the researcher-as-bricoleur metaphor (Lévi-Strauss, 1962; Denzin & Lincoln, 2000), Qualia Coding treats coding as an act of interpretation, not management. Data stays as portable JSON in the researcher's vault — local-first, auditable, zero vendor lock-in.
>
> **Implications:** This presentation demonstrates how embedding QDA within a PKM system changes the relationship between systematic coding and emergent theorizing, and discusses implications for mixed methods pedagogy and tool accessibility.

### Paper Framing (for FQS or TQR)

**Title:** "Coding as Thinking: Design Principles for Qualitative Data Analysis Inside a Personal Knowledge Management System"

**Argument (not a feature list):**
1. Current CAQDAS tools separate coding from thinking (Okonkwo: "In Atlas.ti, I'm managing the project... In Obsidian, I'm just thinking with the data")
2. The researcher-as-bricoleur (Lévi-Strauss) needs a workshop where diverse materials become coherent understanding
3. Crossover mixed analysis (Onwuegbuzie, 1+1=1) demands tools that handle qual and quant data interdependently
4. The asymmetry between quantitization and qualitization tools is an opportunity (Foundations.md call to developers)
5. Qualia Coding's design principles: coding-as-thinking, local-first, 7-engine integration, built-in analytics
6. Implications for mixed methods pedagogy and tool accessibility

**Target length:** 5,000-8,000 words (FQS) or 3,000-5,000 words (TQR methods note).
