## SIMILAR Gaps -- Detailed Review

### Gap 11: Data-Linked Network/Concept Mapping

**Overlap with Roadmap #12 (Research Board enhancements): 65%**

The Research Board exists as a freeform Fabric.js canvas, but it lacks the data-linked network views that ATLAS.ti and MAXQDA's MAXMaps offer. In those tools, network views are dynamically generated from code relationships and can be navigated back to source data. The Research Board is currently a manual canvas.

**Recommendation:** Enhance Research Board roadmap item to explicitly include data-linked network generation (auto-layout from co-occurrence data, clickable nodes that navigate to source). This would close the gap without a new feature.

### Gap 12: Joint Displays (Mixed Methods)

**Overlap with Roadmap #8 (Cross-source) + #9 (Code x Metadata): 60%**

MAXQDA's joint displays are the gold standard for mixed methods -- structured views combining qualitative codes with quantitative variables in a single table. Qualia's Analytics ViewModes cover some of this (Doc-Code Matrix, Chi-Square Tests) but lack the specific "joint display" format that Creswell's framework prescribes.

**Recommendation:** Add "Joint Display" as a named ViewMode within Analytics, combining Cross-source (#8) and Code x Metadata (#9) in MAXQDA's format. Methodological naming matters for academic discoverability.

### Gap 13: Dictionary-Based Text Analysis

**Overlap with Text Statistics + Word Cloud: 55%**

MAXQDA's MAXDictio provides dictionary-based coding (apply predefined word lists as codes), KWIC (Key Word In Context), word trees, and lemmatization. Qualia has Word Cloud and basic Text Statistics but no dictionary-based coding workflow.

**Recommendation:** WAIT. Dictionary-based analysis is a specialized feature. Word Cloud and Text Statistics cover the 80% case. Consider as a future Analytics enhancement if demand emerges.
