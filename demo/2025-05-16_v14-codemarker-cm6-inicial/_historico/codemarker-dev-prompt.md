Prompt para Desenvolvimento do Plugin CodeMarker para Obsidian

Estou desenvolvendo um plugin chamado "CodeMarker" para o Obsidian que permite fazer codificação qualitativa de texto (similar ao MAXQDA, Atlas.ti, NVivo) mas open source e integrado ao Obsidian. Preciso que você me guie através de todas as etapas de desenvolvimento, desde a configuração do ambiente até a implementação das funcionalidades.

Sobre o plugin:

Objetivo: Marcar trechos de texto para análise qualitativa sem modificar o documento original

Funcionalidade principal: Decorações visuais no editor que não alteram o texto, com metadados armazenados separadamente

Interação: Menu contextual ao passar o mouse, alças de redimensionamento, gestão de sobreposições

O que preciso:

Solução de problemas em ambiente de desenvolvimento para plugins Obsidian (Node.js, npm, TypeScript e CodeMirror)

Estrutura inicial do projeto com todos os arquivos necessários

Implementação passo a passo começando com o básico (marcar texto) e avançando para features complexas

Testes e verificações para garantir que cada componente funciona corretamente

Exemplos de código comentados para eu entender cada parte da implementação

Progressão de desenvolvimento:

Setup do ambiente e estrutura do plugin

Sistema básico de marcação com armazenamento separado

Menu flutuante ao passar o mouse sobre marcações

Alças para redimensionar marcações

Gestão de códigos sobrepostos

Começamos do zero, então precisamos incluir absolutamente todos os passos, desde instalar Node.js até testar o plugin no Obsidian. Quero implementar aos poucos, então vamos focar primeiro em fazer a parte mais básica funcionar (marcar texto e visualizar) antes de avançar para as funcionalidades mais complexas.
